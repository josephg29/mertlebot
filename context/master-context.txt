# MERTLEBOT - MASTER CONTEXT FILE

## 🎯 WHAT IS MERTLEBOT?

**mertlebot** is an AI-powered electronics project generator that transforms natural language descriptions into complete, build-ready electronics projects. It's like having an electronics engineer, wiring diagram designer, and code generator all in one tool.

### Core Function
```
User Input: "Create a temperature-controlled fan"
↓
mertlebot Processing
↓
Output:
1. Interactive wiring diagram (wiregen)
2. Complete parts list with specifications
3. Ready-to-flash Arduino code
4. Step-by-step build instructions
5. Optional: Browser-based circuit simulation
```

### Live Demo Concept
Imagine typing "motion-sensing alarm" and getting:
- A professional wiring diagram showing exactly how to connect the PIR sensor, buzzer, and Arduino
- A shopping list: "1x PIR sensor (HC-SR501), 1x Piezo buzzer, 1x Arduino Uno..."
- Complete Arduino code with comments
- Step-by-step assembly instructions
- A clickable simulation to test it in browser

## 🔌 WIREGEN VISUALIZATION ENGINE

### What It Does
Turns JSON data into professional wiring diagrams that would normally require manual CAD work.

### Component Library (10+ Types)
```
Basic:        LED, Resistor, Button, Buzzer
Sensors:      DHT22 (temp/humidity), HC-SR04 (ultrasonic)
Actuators:    ServoMotor
Displays:     OLED Display, LCD I2C
Input:        Potentiometer
```

### Board Support (4 Types)
- Arduino Uno (most common)
- Arduino Mega (more pins)
- Arduino Nano (compact)
- ESP32 DevKit V1 (WiFi/Bluetooth)

### Interactive Features
- **Zoom & Pan**: Navigate complex diagrams
- **Component Selection**: Click to highlight connections
- **Wire Routing**: Automatic right-angle paths
- **De-overlapping**: Clean diagram layout
- **Real-time Updates**: Live rendering as AI generates

### Technical Implementation
- **File**: `WiringCanvas.svelte` (669 lines)
- **Format**: SVG with JavaScript interactivity
- **Canvas**: 960×640 pixels
- **Coordinates**: Pixel-based placement system
- **Routing**: Orthogonal (Manhattan) wire paths

### Example Wiregen JSON
```json
{
  "board": { "type": "arduino-uno", "x": 50, "y": 240 },
  "components": [
    { "id": "led1", "type": "led", "x": 300, "y": 150, "props": { "color": "red" } }
  ],
  "wires": [
    { "color": "yellow", "path": [[92,230],[92,205],[430,205]], "label": "D13" }
  ],
  "componentList": [ { "label": "Red LED", "qty": 1 } ]
}
```

## 🤖 LLM INTEGRATION SYSTEM

### Dual-Provider Architecture
**Anthropic Claude** (Higher cost, excellent quality):
- Models: Claude Haiku (fast), Claude Sonnet (smart)
- Key format: `sk-ant-abc123...`
- Streaming: SDK built-in

**DeepSeek** (Lower cost, very good quality):
- Model: deepseek-chat
- Key format: `sk-123456...`
- Streaming: SSE (OpenAI-compatible)

### LLM Service Abstraction (`llm-service.js` - 363 lines)
```javascript
// Same interface for both providers
class LLMService {
  async generateStream(options)     // Project generation
  async classifyIntent(prompt)      // BUILD vs CHAT
  async clarify(prompt, questions)  // Question selection
  async simulate(prompt, guide)     // Wokwi simulation
  validateApiKey(key)               // Format validation
  getProviderInfo()                 // UI metadata
}
```

### Key Methods
1. **`generateStream()`** - Main project generation with real-time streaming
2. **`classifyIntent()`** - Determines if user wants to BUILD or just CHAT
3. **`clarify()`** - Asks relevant questions before generating
4. **`simulate()`** - Creates Wokwi browser simulations

## 🌐 API ENDPOINTS

### 1. `/api/generate` (179 lines)
**Purpose**: Main project generation
**Input**: Project description, skill level, age
**Output**: Streaming response with wiregen JSON, parts, code, steps
**Features**: Intent classification, skill context, dual-provider routing

### 2. `/api/clarify` (34 lines)
**Purpose**: Question refinement
**Input**: Project description, available questions
**Output**: 0-5 relevant clarification questions
**Timeout**: 3 seconds with regex fallback

### 3. `/api/simulate` (75 lines)
**Purpose**: Circuit simulation generation
**Input**: Project description and guide
**Output**: Wokwi simulation URL + Arduino sketch
**Integration**: Direct Wokwi API communication

### 4. `/api/key` (39 lines)
**GET**: Returns configuration status
**POST**: Saves API key and provider preference
**Validation**: Provider-specific format checking

## 📝 PROMPT ENGINEERING SYSTEM

### File: `prompts.js` (324 lines, 6 sections)

1. **SYSTEM** - Main build guide prompt with strict wiregen JSON format
2. **SKILL_CONTEXT** - Adjustments for 5 skill levels
3. **CHAT_SYSTEM** - Conversational mode prompt
4. **CLARIFY_SYSTEM** - Question selection logic
5. **SIM_PROMPT** - Wokwi simulation generation
6. **QUESTION_CATALOG** - 20+ pre-defined clarification questions

### Strict Output Format
```
# [Project Name]

## WIRING
```wiregen
{JSON diagram with exact coordinates}
```

## PARTS
- [qty]x [Part Name] -- [specifications]

## CODE
```cpp
[Complete, ready-to-flash Arduino code]
```

## STEPS
1. [Step-by-step assembly instructions]
2. [Testing and verification]
3. [Troubleshooting tips]
```

### Key Prompt Features
- Enforces consistent JSON structure for wiregen
- Includes coordinate placement for components
- Provides age-appropriate explanations
- Maintains professional tone across skill levels

## ⚙️ CONFIGURATION SYSTEM

### File: `config.json`
```json
{
  "apiKey": "",           // User's LLM API key
  "provider": "anthropic", // anthropic or deepseek
  "theme": "solder",      // UI theme
  "skill": "MONKEY",      // Skill level
  "age": 25,             // Explanation complexity
  "fontSize": 16         // Base font size
}
```

### Management: `config.js`
- `loadConfig()` - Reads from `config.json`
- `saveConfig(cfg)` - Writes to `config.json`
- `getApiKey()` - Returns key from config or environment
- `getProvider()` - Returns current provider

2. **Describe**: Types project idea (e.g., "plant watering system")
