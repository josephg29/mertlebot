# Mertlebot Application - Technical Context Only

## APPLICATION OVERVIEW
- **Name**: mertlebot (mertle.bot)
- **Tagline**: "describe it. build it."
- **Function**: Electronics project generator - converts natural language descriptions into complete build guides with wiring diagrams
- **Current Deployment**: Single-instance, file-based configuration

## TECHNICAL ARCHITECTURE

### Frontend Stack
- **Framework**: Svelte 5
- **Meta-framework**: SvelteKit
- **Build Tool**: Vite
- **Styling**: CSS with custom properties (CSS variables)
- **Key Component**: Wiregen visualization engine (client-side SVG)

### Backend Stack
- **Runtime**: Node.js
- **API Framework**: SvelteKit server routes
- **Configuration**: JSON file (`config.json`)
- **Streaming**: Server-Sent Events (SSE)

### LLM Integration
- **Providers Supported**: Anthropic Claude, DeepSeek
- **Abstraction Layer**: `llm-service.js` (363 lines)
- **Interface Methods**: `generateStream()`, `classifyIntent()`, `clarify()`, `simulate()`
- **Key Validation**: Provider-specific format checking

## WIREGEN VISUALIZATION ENGINE

### Component Library
- **Basic Components**: LED, Resistor, Button, Buzzer
- **Sensors**: DHT22 (temperature/humidity), HC-SR04 (ultrasonic)
- **Actuators**: ServoMotor
- **Displays**: OLED Display, LCD I2C
- **Input**: Potentiometer

### Board Support
- Arduino Uno
- Arduino Mega
- Arduino Nano
- ESP32 DevKit V1

### Technical Implementation
- **File**: `WiringCanvas.svelte` (669 lines)
- **Technology**: SVG with JavaScript interactivity
- **Features**: Zoom, pan, component selection, wire routing
- **Wire Routing**: Orthogonal (right-angle) paths with de-overlapping
- **Coordinate System**: Pixel-based placement (960×640 canvas)

## API ENDPOINTS

### `/api/generate` (179 lines)
- **Method**: POST
- **Purpose**: Main project generation
- **Response**: Streaming SSE
- **Features**: Intent classification, skill/age context, dual-provider support

### `/api/clarify` (34 lines)
- **Method**: POST
- **Purpose**: Question clarification selection
- **Timeout**: 3 seconds
- **Fallback**: Regex filtering if LLM times out

### `/api/simulate` (75 lines)
- **Method**: POST
- **Purpose**: Wokwi simulation generation
- **Integration**: Direct Wokwi API communication
- **Output**: Embeddable simulation URL + Arduino sketch

### `/api/key` (39 lines)
- **GET**: Returns configuration status
- **POST**: Saves API key and provider preference
- **Validation**: Provider-specific key format checking

- **Caching**: Local storage for preferences
- **Lazy Loading**: Component libraries loaded as needed
